# 修复分类管理 403 错误

## 问题
访问分类管理接口时返回 403 Forbidden 错误。

## 原因
这通常是权限验证问题，可能的原因：
1. JWT Token 中没有正确的 ADMIN 角色信息
2. 用户没有重新登录，使用的是旧的 Token
3. 权限配置问题

## 解决步骤

### 步骤1：重新登录（最重要）
**必须重新登录**以获取包含正确角色信息的新 JWT Token：

1. **退出登录**
   - 点击右上角用户头像 → 退出登录
   - 或者清除浏览器 localStorage

2. **清除浏览器缓存**
   - 打开开发者工具（F12）
   - Application → Local Storage → 清除所有数据

3. **重新登录**
   - 用户名：`admin`
   - 密码：`123456`

### 步骤2：验证 Token 中的角色
在浏览器控制台执行：
```javascript
console.log('Role:', localStorage.getItem('role'))
console.log('Token:', localStorage.getItem('token')?.substring(0, 50) + '...')
```

应该显示：
- `Role: ADMIN`（注意是大写）

### 步骤3：检查后端日志
查看后端控制台，确认：
- JWT Token 解析成功
- 角色信息正确：`ROLE_ADMIN`
- 没有权限相关的错误

### 步骤4：如果还是不行
检查数据库中的用户角色：
```sql
USE agricultural_db;
SELECT username, role FROM users WHERE username = 'admin';
```

应该显示 `role = 'ADMIN'`（大写）

## 快速修复

如果上述步骤都不行，请：

1. **完全清除浏览器数据**
   - 清除所有缓存和 localStorage
   - 关闭浏览器重新打开

2. **重启后端服务**
   ```bash
   cd backend
   # 停止服务（Ctrl+C）
   mvn spring-boot:run
   ```

3. **重新登录**
   - 使用 admin / 123456 登录

4. **访问分类管理**
   - 应该可以正常访问了

## 验证

重新登录后，访问分类管理页面应该：
- ✅ 可以正常加载分类列表
- ✅ 可以添加、编辑、删除分类
- ✅ 没有 403 错误
