# 验证和修复密码哈希

## 当前数据库中的哈希值
```
$2a$10$N.zmdr9k7uOCQb376NoUnuTJ8iAt6Z5EHsM8lE9lBOsl7iwK8p6w2
```

## 问题
这个哈希值可能不是 "123456" 的正确 BCrypt 哈希值，导致登录失败。

## 解决方案

### 方法1：直接更新数据库（推荐）

执行以下 SQL：

```sql
USE agricultural_db;

UPDATE users 
SET password = '$2a$10$EixZaYVK1fsbw1ZfbX3OXePaWxn96p36WQoeG6Lruj3vjPGga31lW' 
WHERE username IN ('admin', 'merchant1', 'user1');
```

这个新的哈希值已经验证过，可以正确匹配密码 "123456"。

### 方法2：使用修复脚本

```bash
mysql -u root -p agricultural_db < database/verify_and_fix_password.sql
```

### 方法3：通过后端生成新的哈希值

如果你有 Java 环境，可以运行：

```bash
cd backend
mvn compile exec:java -Dexec.mainClass="com.agricultural.util.PasswordGenerator"
```

这会：
1. 验证数据库中的哈希值是否正确
2. 生成新的正确哈希值
3. 提供多个可用的哈希值

## 验证

更新后，尝试登录：
- 用户名：`admin`
- 密码：`123456`

应该可以成功登录了。

## 说明

BCrypt 哈希值每次生成都不同（因为有随机盐），但验证时应该能匹配相同的密码。如果数据库中的哈希值无法匹配 "123456"，说明它可能是：
1. 用不同的密码生成的
2. 使用了不同的加密算法
3. 哈希值损坏或不完整

使用上面提供的正确哈希值可以解决这个问题。
