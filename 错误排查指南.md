## 常见错误及解决方案

### 错误1：Bean 创建失败 / 循环依赖
**症状**：启动时出现 `Error creating bean` 或 `Circular reference`

**解决方案**：
1. 确保已删除 `CorsConfig.java`
2. 重启后端服务

### 错误2：404 Not Found
**症状**：前端请求返回 404

**检查清单**：
- [ ] 后端是否已启动（检查 http://localhost:8080）
- [ ] 前端是否已启动（检查 http://localhost:3000）
- [ ] 后端控制台是否有错误信息
- [ ] 浏览器 Network 标签中请求的 URL 是否正确

**解决方案**：
```bash
# 重启后端
cd backend
mvn spring-boot:run

# 重启前端
cd frontend
npm run dev
```

### 错误3：CORS 错误
**症状**：浏览器控制台显示 CORS 相关错误

**解决方案**：
- 已修复，确保 SecurityConfig 中的 CORS 配置正确
- 重启后端服务

### 错误4：数据库连接失败
**症状**：`Communications link failure` 或 `Access denied`

**解决方案**：
1. 检查 MySQL 是否启动
2. 检查 `application.yml` 中的数据库配置：
   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://localhost:3306/agricultural_db
       username: root
       password: your_password
   ```
3. 确保数据库已创建：
   ```bash
   mysql -u root -p < database/init.sql
   ```

### 错误5：401 Unauthorized
**症状**：登录后仍然返回 401

**检查**：
- JWT Token 是否正确生成
- 请求头中是否包含 `Authorization: Bearer <token>`
- Token 是否过期

### 错误6：500 Internal Server Error
**症状**：服务器内部错误

**检查**：
1. 查看后端控制台的完整错误堆栈
2. 检查数据库表是否已创建
3. 检查实体类字段是否与数据库表结构匹配

## 调试步骤

### 1. 检查后端日志
查看后端控制台输出，寻找：
- 启动成功的消息
- 数据库连接信息
- 任何红色错误信息

### 2. 测试后端 API
使用 curl 或 Postman 直接测试后端：

```bash
# 测试登录接口
curl -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"user1","password":"123456"}'
```

### 3. 检查前端网络请求
1. 打开浏览器开发者工具（F12）
2. 切换到 Network 标签
3. 尝试登录
4. 查看请求的：
   - URL（应该是 `http://localhost:3000/api/auth/login`）
   - 状态码
   - 响应内容

### 4. 检查控制台错误
查看浏览器控制台（Console 标签）中的：
- JavaScript 错误
- 网络错误
- 任何红色错误信息

## 快速修复命令

```bash
# 1. 停止所有服务（Ctrl+C）

# 2. 清理并重新编译后端
cd backend
mvn clean compile

# 3. 启动后端
mvn spring-boot:run

# 4. 在新终端启动前端
cd frontend
npm run dev
```
