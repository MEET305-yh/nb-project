# 修复密码问题

## 问题原因
数据库中的密码哈希值可能不正确，导致无法登录。

## 解决方案

### 方法1：执行修复脚本（推荐）

```bash
mysql -u root -p agricultural_db < database/fix_passwords.sql
```

这个脚本会：
1. 删除旧的测试用户
2. 插入新的测试用户，使用正确的 BCrypt 哈希值

### 方法2：手动更新数据库

连接到 MySQL 并执行：

```sql
USE agricultural_db;

-- 更新所有用户的密码为正确的 BCrypt 哈希值（密码: 123456）
UPDATE users SET password = '$2a$10$EixZaYVK1fsbw1ZfbX3OXePaWxn96p36WQoeG6Lruj3vjPGga31lW' 
WHERE username IN ('admin', 'merchant1', 'user1');
```

### 方法3：通过注册新用户

如果上述方法不行，可以通过前端注册一个新用户，然后手动修改数据库中的角色。

## 测试账号

修复后，可以使用以下账号登录：

- **角色**: ADMIN

- **用户名**: merchant1

- **用户名**: user1

2. 点击登录
3. 输入用户名：admin
4. 输入密码：123456
5. 应该能成功登录
