# 权限问题排查和修复

## 问题
访问管理端 API 时返回 403 Forbidden 错误。

## 可能的原因

1. **JWT Token 中的角色信息不正确**
2. **用户没有重新登录，使用的是旧的 Token**
3. **数据库中的用户角色字段不正确**

## 解决步骤

### 步骤1：确认用户角色
检查数据库中 admin 用户的角色是否正确：

```sql
USE agricultural_db;
SELECT id, username, role FROM users WHERE username = 'admin';
```

应该显示 `role = 'ADMIN'`（注意是大写）

### 步骤2：重新登录
**重要**：如果之前已经登录过，需要：
1. 退出登录
2. 重新使用 admin / 123456 登录
3. 这样会生成新的 JWT Token，包含正确的角色信息

### 步骤3：检查 Token 中的角色
在浏览器控制台执行：
```javascript
// 查看当前用户信息
console.log(localStorage.getItem('role'))
// 应该显示 'ADMIN'
```

### 步骤4：验证后端权限配置
确保：
1. 后端已重启
2. SecurityConfig 中已启用 @EnableMethodSecurity
3. Controller 上使用了 @PreAuthorize("hasRole('ADMIN')")

## 快速修复

如果上述步骤都不行，请执行：

1. **清除浏览器缓存和 localStorage**
   - 打开浏览器开发者工具
   - Application/存储 → Local Storage → 清除所有数据

2. **重新登录**
   - 使用 admin / 123456 登录

3. **重启后端服务**
   ```bash
   cd backend
   mvn spring-boot:run
   ```

4. **检查后端日志**
   - 查看是否有权限相关的错误信息
   - 查看 JWT 解析是否成功

## 如果还是不行

请检查：
1. 后端控制台是否有错误日志
2. 浏览器 Network 标签中请求的响应内容
3. 确认 JWT Token 是否正确传递（Authorization 头）
