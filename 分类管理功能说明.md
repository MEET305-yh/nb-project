# 分类管理功能说明

## 功能概述

在管理端新增了分类管理功能，可以对商品分类进行完整的增删改查操作。

## 数据库变更

### 新增分类表

执行以下 SQL 创建分类表：

```bash
mysql -u root -p agricultural_db < database/add_categories_table.sql
```

或者手动执行：

```sql
USE agricultural_db;

CREATE TABLE IF NOT EXISTS categories (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL UNIQUE COMMENT '分类名称',
    description VARCHAR(200) COMMENT '分类描述',
    sort_order INT DEFAULT 0 COMMENT '排序顺序',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    deleted INT DEFAULT 0 COMMENT '删除标记: 0-未删除, 1-已删除',
    INDEX idx_name (name),
    INDEX idx_sort (sort_order)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='分类表';

-- 插入初始分类数据
INSERT INTO categories (name, description, sort_order, deleted) VALUES
('水果', '新鲜水果', 1, 0),
('蔬菜', '绿色蔬菜', 2, 0),
('粮食', '优质粮食', 3, 0),
('其他', '其他农产品', 4, 0);
```

## 功能特性

### 1. 分类列表
- ✅ 分页显示
- ✅ 支持按分类名称搜索
- ✅ 显示分类名称、描述、排序、创建时间

### 2. 添加分类
- ✅ 分类名称（必填，唯一）
- ✅ 分类描述（可选）
- ✅ 排序顺序（数字越小越靠前）

### 3. 编辑分类
- ✅ 修改分类名称、描述、排序
- ✅ 检查分类名称唯一性

### 4. 删除分类
- ✅ 软删除（设置 deleted=1）
- ✅ 删除提示确认

## 访问方式

1. 登录管理员账号（admin / 123456）
2. 进入管理后台：`http://localhost:3000/admin`
3. 点击左侧菜单"分类管理"

## API 接口

### 获取分类列表
- `GET /admin/categories` - 分页获取分类列表
- `GET /admin/categories/all` - 获取所有分类（不分页）

### 获取分类详情
- `GET /admin/categories/{id}` - 获取单个分类详情

### 创建分类
- `POST /admin/categories` - 创建新分类

### 更新分类
- `PUT /admin/categories/{id}` - 更新分类信息

### 删除分类
- `DELETE /admin/categories/{id}` - 删除分类（软删除）

## 注意事项

1. **分类名称唯一性**：系统会检查分类名称是否已存在
2. **删除操作**：删除分类是软删除，不会真正从数据库删除
3. **商品关联**：删除分类不会影响已使用该分类的商品，商品的分类字段保持不变
4. **排序功能**：可以通过设置 `sortOrder` 来控制分类的显示顺序
