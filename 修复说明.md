## 验证步骤

1. **检查后端是否启动**
   - 访问：http://localhost:8080/auth/login（应该返回 405 或 JSON 错误，说明服务已启动）
   - 或者查看后端控制台，确认没有错误

2. **检查前端是否启动**
   - 访问：http://localhost:3000
   - 应该能看到首页

3. **测试登录功能**
   - 使用测试账号：`user1` / `123456`
   - 或者：`admin` / `123456`

## 当前配置说明

### 后端
- 运行在：`http://localhost:8080`
- API 路径示例：`/auth/login`, `/products/public/list`

## 如果还有问题

1. **检查后端日志**
   - 查看是否有数据库连接错误
   - 查看是否有端口占用问题

2. **检查前端控制台**
   - 打开浏览器开发者工具（F12）
   - 查看 Network 标签，检查请求的 URL 和响应

3. **检查数据库**
   - 确保 MySQL 已启动
   - 确保数据库 `agricultural_db` 已创建
   - 确保已执行初始化脚本

4. **测试后端 API**
   - 使用 Postman 或 curl 直接测试：
   ```bash
   curl -X POST http://localhost:8080/auth/login \
     -H "Content-Type: application/json" \
     -d '{"username":"user1","password":"123456"}'
   ```

## 常见错误及解决方案

### 错误1：404 Not Found
- **原因**：后端未启动或路径配置错误
- **解决**：检查后端是否运行在 8080 端口

### 错误2：CORS 错误
- **原因**：跨域配置不正确
- **解决**：已修复，重启后端即可

### 错误3：数据库连接失败
- **原因**：MySQL 未启动或配置错误
- **解决**：检查 `application.yml` 中的数据库配置

### 错误4：401 Unauthorized
- **原因**：JWT Token 无效或过期
- **解决**：重新登录获取新 Token
